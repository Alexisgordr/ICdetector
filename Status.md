# ICdetection Status

## v2.0 Fully Frozen

The v2.0 fix phase is now finished.

ICdetection is entering a field data collection phase. For the next 2 to 3 months, the main goal is to collect real-world antenna history and observe how the app behaves across different places, routes, operators, and radio conditions.

Because several important stability and data-integrity fixes have been applied, I recommend clearing the local database and starting fresh before beginning this collection phase. This helps ensure that the new baseline is built only from clean data.

No more fixes or feature changes are planned until v2.1, unless a truly critical bug is discovered.

I have done my best to make this version as stable, honest, and useful as possible. This is still a defensive anomaly detector, not a guaranteed IMSI-catcher detector, and real-world data is necessary to understand its strengths and limitations.

If you find a bug, unexpected behavior, or something that does not look right, please open an issue and report it with as much detail as possible.

Thank you very much to everyone testing the app. I really appreciate your patience during this data collection phase.
